## 📂 目录分类说明

### 1. 核心文档层（项目根目录）

| 文件 | 大小 | 说明 |
|------|------|------|
| README.md | ~7 KB | 项目主入口文档，包含概述、快速开始、使用指南 |
| CHANGELOG.md | ~2.5 KB | 版本变更历史，记录每次更新的详细内容 |
| API文档.md | ~12.6 KB | 服务接口文档，包含配置、连接、示例 |
| 代码注释说明.md | ~13.6 KB | 代码注释规范和结构说明 |
| 文档索引.md | ~5.9 KB | 文档快速导航，按主题分类索引 |
| 项目结构说明.md | 本文件 | 完整的目录结构说明 |

### 2. Docker 模块层（dockers/）

**配置文件**：
- `docker-compose.yml` - 统一的服务编排配置
- `run_env.sh` - 一键启动所有服务

**文档文件**：
- `Docker笔记-常用命令.md` - Docker 命令参考手册
- `Docker笔记-c++开发环境.md` - C++ 开发环境配置教程

**环境子目录**：
每个环境子目录包含：
- `Dockerfile` - 镜像构建文件
- `docker-compose.yml` - 独立配置（可选）
- `docker-entrypoint.sh` - 启动脚本
- `build.sh` - 构建脚本
- `run.sh` - 运行脚本
- 配置文件目录（如 `conf/`）

### 3. Shell 模块层（shell/）

**Oh-My-Zsh 子目录**：
- `install.sh` - 安装脚本（8.4 KB，含详细中文注释）
- `zshrc` - 配置文件（7.0 KB，含详细中文注释）
- `最好用的Shell(zsh).md` - 使用指南（6.4 KB）

**brew 子目录**：
- `list-brew-packages-by-date.sh` - 列出包
- `uninstall-brew-by-date.sh` - 卸载包

### 4. Git 模块层（git/）

- `配置自己的git忽略文件.md` - Git ignore 模板说明

### 5. Vim 模块层（vim/）

- `配置自己的vim.md` - Vim 配置指南
- `vimrc1` - 配置文件版本1（11 KB）
- `vimrc2` - 配置文件版本2（10 KB）

### 配置文件命名

| 类型 | 命名模式 | 示例 |
|------|----------|------|
| Docker 镜像构建 | `Dockerfile` | 所有环境目录 |

## 📝 文件编辑建议

### 新增 Docker 环境

1. 在 `dockers/` 下创建新目录
2. 创建 `Dockerfile`
3. 创建 `build.sh` 和 `run.sh`
4. （可选）创建 `docker-compose.yml`
5. （可选）创建 `conf/` 配置目录
6. 更新主 `dockers/docker-compose.yml`
7. 更新 `README.md` 和 `API文档.md`

### 新增 Shell 脚本

1. 在 `shell/` 或子目录下创建 `.sh` 文件
2. 添加文件头注释（参考 `install.sh`）
3. 添加函数注释（参数、返回值）
4. 更新相关文档

### 新增文档

1. 在项目根目录或模块目录下创建 `.md` 文件
2. 遵循现有文档格式
3. 更新 `文档索引.md`
4. 更新 `README.md`（如需要）

## ⚠️ 注意事项

1. **文件路径引用**
   - 绝对路径使用项目根目录作为基准
   - 相对路径使用 `./` 表示当前目录
   - 跨目录引用使用 `../` 表示上级目录

2. **配置文件冲突**
   - `shell/Oh-My-Zsh/zshrc` 与 `~/.zshrc` 可能冲突
   - 使用时注意备份现有配置

3. **Docker 端口冲突**
   - 确保宿主机端口未被占用
   - 修改 `docker-compose.yml` 中的端口映射

4. **权限问题**
   - Shell 脚本需要执行权限：`chmod +x script.sh`
   - Docker 数据卷需要适当的读写权限
